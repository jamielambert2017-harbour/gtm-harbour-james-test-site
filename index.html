The GTM Playground

A safe sandbox for tags, triggers, variables & the dataLayer — built by James Humphreys
Checking for a container…

Point your own container at this page

  1. In Google Tag Manager, create a free Web container (or open one you already have).
  2. Copy its ID — it looks like GTM-XXXXXXX.
  3. Paste it below and load. The page reloads with ?id=GTM-XXXXXXX and injects your container.

No container yet? You can still click around — every button below pushes to the dataLayer, and the inspector (bottom-right) shows it. You just won't see anything fire in GTM Preview until a container is loaded.

01

What's already in the dataLayer

Before GTM even loaded, this page pushed a set of values. Open the inspector (bottom-right) and hit View raw dataLayer to see them. Two flavours are mixed in on purpose:

custom_dl_param1 / param2 / param3 / custom_dl_objectobvious teaching names — easy to spot in Preview
page_type · page_category · user_login_state · visitor_typethe kind of context a real site pushes

Try this → Build a Data Layer Variable named custom_dl_param1, then a second one for page_type. Preview the page and confirm both pick up their values.

Engagement & custom events

Non-ecommerce events — the everyday stuff. The last one pushes the obvious teaching params so you can practise reading them off a custom event.

04

Form submission

Fill it in and submit. Nothing is sent anywhere — on submit the page pushes a newsletter_signup event carrying the field values, then resets.

Try this → A real form usually has no tidy dataLayer event. Practise both: (a) the easy way — use the newsletter_signup event this page gives you; (b) the hard way — build a trigger on GTM's built-in Form Submission listener instead, and notice the difference.

05

06

DOM scraping playground

Sometimes the value you want is on the page but never in the dataLayer. That's when you scrape the DOM. Each row below shows a value and the exact selector to grab it with a DOM Element or Custom JavaScript variable.

Price — £29.99ID: #dom-price · or attribute data-price on #product-price
Product name — Harbour.Space HoodieClass: .dom-title
SKU (hidden in an attribute)#product-name → data-item-id ("SKU_HOODIE_01")

A value hidden from view#hidden-value (display:none, but still scrapable)
scraped_from_hidden_dom

Try this → Build a DOM Element variable using CSS Selector #dom-price (leave attribute blank to grab the text). Then a tougher one: Custom JavaScript variable that reads document.getElementById('product-price').dataset.price and returns it as a number.

07

Lookup & RegEx table practice

Pick a region. The page pushes a region_selected event with a raw region code — your job in GTM is to map that code to something friendlier.

Try this → Build a Lookup Table variable: input = your region Data Layer Variable, mapping th → Thailand, gb → United Kingdom, etc. Then rebuild it as a RegEx Table and add a default value for anything unmatched.
