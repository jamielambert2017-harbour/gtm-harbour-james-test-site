Checking for a container…

Point your own container at this page

  1. In Google Tag Manager, create a free Web container (or open one you already have).
  2. Copy its ID — it looks like GTM-XXXXXXX.
  3. Paste it below and load. The page reloads with ?id=GTM-XXXXXXX and injects your container.

No container yet? You can still click around — every button below pushes to the dataLayer, and the inspector (bottom-right) shows it. You just won't see anything fire in GTM Preview until a container is loaded.

Consent state (Consent Mode v2)

Before the container even loaded, this page set every consent signal to denied by default. The cookie banner's Accept button calls gtag('consent','update',…) and flips them to granted. Watch these cells change — this is the live consent state your tags must respect.

Consent group → Your job: make the Google Tag fire only once consent is given. In your tag's Consent Settings → Require additional consent for tag to fire, add analytics_storage. Now Preview: with the banner unanswered the tag is blocked; click Accept and watch it fire. Stretch: leave the tag on All Pages and observe how Consent Mode changes its behaviour denied → granted instead of blocking it.

Form group → Use the form in section 04. Build a tag that fires on submit and captures the field values as variables. Ecommerce group → Use the buttons in section 02. Build event tags on add_to_cart, begin_checkout and purchase.

07

Lookup & RegEx table practice

Pick a region. The page pushes a region_selected event with a raw region code — your job in GTM is to map that code to something friendlier.

Try this → Build a Lookup Table variable: input = your region Data Layer Variable, mapping th → Thailand, gb → United Kingdom, etc. Then rebuild it as a RegEx Table and add a default value for anything unmatched.

08

QA Lab

Your dev team shipped eight ecommerce pushes for review. Fire each one, inspect what actually lands (Preview / DebugView / the inspector, bottom-right), and validate it against the spec you've been given. Which pushes are correct? Which are broken — and what would each fault break in GA4?

The game → 1 point for spotting a fault, a bonus point for explaining what it breaks in reporting. Not every push is broken — verify, don't assume.
